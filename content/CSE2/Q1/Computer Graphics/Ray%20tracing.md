If at some point this light ray this light source hits the camera again, we will record the light and produce an image.</p><p><br></p><p>But, as visible in the following image, the probability that this light will hit the camera is pretty low, as the camera is only one point in space:</p><p><img src="https://i.imgur.com/qLbWw0e.png" width="369"></p><p><em>Image taken from slides by Elmar Eisemann</em></p><p><br></p><p>So why does this work in real life? In real life, there are many, many more light rays per second, of which many will hit your eye. We can solve this by starting ray tracing at camera and check for light at each hit point:</p><p><img src="https://i.imgur.com/xKo2zIc.png" width="457"></p><p><em>Image taken from slides by Elmar Eisemann</em></p><p><br></p><p>In Whitted-Style ray tracing, this image illustrates the following:</p><ul><li>You start with rays at your camera and check what happens</li><li>On diffuse surfaces, we are done</li><li>On reflective surfaces, we shoot additional rays</li><li>On the end of these rays, we check again</li><li>Etc</li></ul><p><br></p><p>At some point you will end up with a final result for every pixel. But, this is a very big simplification with some downsides:</p><ul><li>When do we stop?</li><li>You only shoot a few rays so reflections will be poor</li><li>At diffuse surfaces we stop if we can't see the light, leading to poor shadows. Normally it accepts light from all directions whereas this doesn't </li></ul><p><br></p><p><strong>Algorithm</strong></p><p>The algorithm described previously will be a recursive algorithm:</p><p><br></p><p>For all pixels:</p><ul><li>Compute <strong>ray trough pixel</strong></li><li>Compute <strong>first hitpoint</strong></li><li>Compute <strong>ray to light </strong>(for shadows) and <strong>shading</strong></li><li>Compute <strong>reflected </strong>and <strong>refracted </strong>ray</li><li><strong>Recursively continue </strong>for both rays</li></ul><p><br></p><p><br></p><p><strong>Point lights</strong></p><p>Point lights are infinitely small and emit light equally in all directions:</p><p><img src="https://i.imgur.com/J5hk8TN.png" width="317"></p><p><em>Image taken from slides by Elmar Eisemann</em></p><p><br></p><p>We want to shade this hitpoint, so we do the shadow test:</p><ul><li>Test if <strong>any object </strong>lies between the hitpoint and light source</li><li>If in shadow skip further computation for this light source</li><li>Else compute my shading and continue</li></ul><p><br></p><p>Then we get a sum of different contributions:</p><ul><li>Radiance from <strong>direct light </strong>(if not in shadow)</li><li>Radiance from <strong>reflected light </strong>(if surface is glossy / specular)</li><li>Radiance from <strong>refracted light </strong>(if surface transmits light)</li></ul><p><br></p><p>Our total radiance (color) at any point is the sum of these.</p><p><br></p><h3 id="in-code">In code</h3><p>The ray tracer has two main function:</p><ul><li><em>trace, </em>which computes a hitpoint of ray with scene</li><li><em>shade, </em>which computes color for a given point in the scene</li></ul><p><br></p><p>These functions recursively call each other. The intersect function is a function to test the ray against scene elements. function Trace(level, ray, &amp;color) {
    if (Intersect(level, ray, max, &amp;hit)) 
       Shade(level, hit, &amp;color);
    else 
       color = backgroundColor;
}


function Shade(level, hit, &amp;color) {
   ComputeDirectLight(hit, &amp;directColor);


   ComputeReflectedRay(hit, &amp;reflectedRay);
   Trace(level + 1, reflectedRay, &amp;reflectedColor); // as we just computed the reflected ray, it might reflect again


   ComputeRefractedRay(hit, &amp;refractedRay);
   Trace(level + 1, refractedRay, &amp;refractedColor); // same as for reflected


   color = directColor + 
           reflection * reflectedColor + 
           transmission * refractedColor;
}
