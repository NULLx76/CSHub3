<h1 id="unsupervised-learning">Unsupervised learning</h1><p>Here is an overview of different machine learning models:</p><p><img src="https://i.imgur.com/MXmplen.png" width="711"></p><p><em>Image taken from slides by Gosia Migut</em></p><p><br></p><p>All of these had in common that the dataset was labeled, but in unsupervised learning we deal with datasets that are unlabeled.</p><p><br></p><p>There are two techniques: dimensionality reduction and clustering</p><p><br></p><h2 id="dimensionality-reduction">Dimensionality reduction</h2><h3 id="why?"><strong>Why?</strong></h3><p>Many datasets exist of many features (dimensions), we can reduce the dimensionality to:</p><ul><li>Make storage or processing of data easier</li><li>(Visual) discovery of hidden structure in the data</li><li>Remove redundant (e.g. duplicate) and noisy features</li><li>Intrinsic dimensionality might smaller</li><li>As dimensionality grows: fewer observations per region</li></ul><p><br></p><p>If you have a 3-dimensional dataset and you can put all points on a plane you can already notice that we can lose a dimension.</p><p><br></p><h3 id="what?"><strong>What?</strong></h3><p>Reducing dimensionality is done by transforming high-dimensional data to data of lower dimensionality, while preserving the structure in the original data as good as possible</p><p><br></p><p>We do this by picking a subset of the original dimensions based on domain knowledge and thus construct a new set of dimensions that are linear combinations of the original dimensions.</p><p><br></p><p>These are often linear techniques, which project the data on a linear subspace of a lower dimensionality (principal component analysis):</p><p><img src="https://i.imgur.com/pGnvAt5.png" width="618"></p><p><em>Image taken from slides by Gosia Migut</em></p><p><br></p><h3 id="principal-component-analysis">Principal component
