<p>In clustering, we want to understand the structure given a cloud of data points in high dimensional data. More formally, given a <strong>set of points</strong>, with a notion of <strong>distance </strong>between points, <strong>group the points </strong>into some number of clusters so that members of a cluster are close to each other and members of different clusters are dissimilar.</p><p><br></p><p>Clustering is hard because it involves many dimensions as almost all pairs of points are at about the same distance when involving tens of thousands of dimensions.</p><p><br></p><p>The distance measure we use, depends on the types of data or how we represent the data:</p><ul><li><strong>Set of words</strong>: Jaccard distance</li><li><strong>Points in space</strong>: Euclidean distnace</li><li><strong>Vector in space of words</strong>: cosine distance</li></ul><p><br></p><p>We can use two methods of clustering:</p><ul><li><strong>Hierarchical</strong></li><li class="ql-indent-1"><strong>Agglomerative</strong> (bottom up); intially, each point is a cluster, and merge the two "nearest" clusters</li><li class="ql-indent-1"><strong>Divisive</strong> (top down); start with one cluster and split it</li><li><strong>Point assignment</strong>: maintain a set of clusters and points belong to the "nearest" cluster</li></ul><p><br></p><h2 id="hierarchical-clustering">Hierarchical clustering</h2><p><a href="https://cshub.nl/post/368910140#clustering" target="_blank">See this post</a>, this will only contain new information not found in that post.</p><p><br></p><p>In non-Euclidean space, a <strong>clustroid </strong>is an exitisting data point "closest" to other points. We use this concept as there is no "average" of two points, we can only about the points themselves.
