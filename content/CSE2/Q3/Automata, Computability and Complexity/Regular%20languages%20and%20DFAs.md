<p>Regular languages, Regular expressions and Finite State Machines (FSMs) have the same expressive power. Every one of these three can be converted into every other one. </p><p><br></p><h1 id="finite-state-machines">Finite State Machines</h1><ul><li>A FSM consists of nodes and edges and is like a directed graph. Each node in the graph is a state and each edge is a transition from one state to another. </li><li>A FSM has one node that is the <strong>starting state</strong> which is the state in which the machine begins (indicated by an arrow pointing to this node). An FSM also has zero or more <strong>accepting states </strong>(indicated by a double circle). When the machine reaches such state, the computation has finished.</li><li>Every finite state machine also has an <strong>alphabet</strong> of symbols used to label edges, e.g. <span class="ql-formula" data-value="\Sigma=\left\{0,1\right\}">﻿<span contenteditable="false"><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi mathvariant="normal">Σ</mi><mo>=</mo><mrow><mo fence="true">{</mo><mn>0</mn><mo separator="true">,</mo><mn>1</mn><mo fence="true">}</mo></mrow></mrow><annotation encoding="application/x-tex">\Sigma=\left\{0,1\right\}</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 0.68333em; vertical-align: 0em;"></span><span class="mord">1</span></span></span></span></span>﻿</span>. </li></ul><p><br></p><h2 id="use-cases">Use cases</h2><h3 id="string-generation"><strong>String generation</strong></h3><p>With a finite state machine, you can generate a sequence of letters (<strong>string</strong>) in an <strong>alphabet</strong> by:</p><ul><li>starting in the starting <strong>state</strong>.</li><li>taking transitions to other states, either in a regular way or at random. Each transition you take is labeled with a letter in the <strong>alphabet</strong> and is added to the generated string.</li><li>only finish up in an accepting state</li></ul><p><br></p><p>In this mode, one is usually interested in the set of strings the machine can generate. This set is also the set of all strings the machine can accept. This set is called the <strong>language</strong> of the FSM.</p><p><br></p><h3 id="string-recognition-/-acceptance"><strong>String recognition / acceptance</strong></h3><p>When the FSM must accept a string, it starts in the <strong>starting state</strong>. For each symbol in the string, it takes the edge out of the current state labelled with the corresponding letter in the <strong>alphabet</strong>. As you traverse the graph based on the input string, two things can happen. Either it ends up in an <strong>accepting state</strong> when the last character has been parsed, or the string ends while the machine hasn't reached an accepting state. In the former case, the string is <strong>accepted</strong> by the FSM. Otherwise the string is <strong>rejected</strong>.</p><p><br></p><h2 id="formal-definition-of-an-fsm">Formal definition of an FSM</h2><div style="white-space: normal;" class="markdown-body"><p>A FSM
