Non-linear classification

Non-linear classification is about separating classes which don't have a straight decision boundary.


Decision trees

If we take the XOR problem: given classes and samples ﻿X=(0,1); (1,0)X=\left(0,1\right);\ \left(1,0\right)X=(0,1); (1,0)﻿ and ﻿Y=(0,0); (1,1)Y=\left(0,0\right);\ \left(1,1\right)Y=(0,0); (1,1)﻿, how would we separate that?


We can use a decision tree:

This doesn't give a single straight line, but a few questions which determine our outcome.


Definition

We have a root node at the top, from which we have branches to decision nodes which have test functions. This goes on until we get to leaves, which classify the sample.


Training

We:

  • split our dataset into subsets
  • are they pure (are all samples positive / negative)?
  • if yes: stop
  • if not: repeat and add more decision nodes


ID3

Deciding how to split is done using the ID3 algorithm: It defines split function: Split(node,{examples})\text{Split}(\text{node}, \{\text{examples}\})Split(node,{examples})

  • Decide which attribute to split on (e.g. AAA)
  • For each value of AAA, create a new child node
  • Split training {examples}\{\text{examples}\}{examples} to child nodes
  • If examples are perfectly classified, stop. Else, iterate over new child nodes using the split function

To avoid overfitting we use pruning: we stop splitting when it is not statistically significant. We can use sub-tree replacement pruning:

  • For each node we pretend to remove the node + all children from the tree and measure performance on the validation set
  • Remove node that result in greatest improvement
  • Repeat until further pruning is harmful

Multi-layer perceptrons

An artificial neural network is inspired by the human brain. It consists out of:

  • Neurons: perceptrons, which are basic processing units
  • Synapses: connections, which connect the perceptrons


So a multi-layer perceptron exists of several layers of connected perceptrons. A single perceptron is a binary classifier which contains the following elements:

  • ﻿xjx_jxj​﻿: an input unit ﻿∈R\in R∈R﻿, ﻿j=1,…,dj=1,\dots,dj=1,…,d﻿
  • ﻿x0x_0x0​﻿: bias (always 1)
  • ﻿wjw_jwj​﻿: connection weight
  • ﻿yyy﻿: output unit

Actually assigning a sample to a class is as easy as choosing the output unit with the highest value: choose ﻿CiC_iCi​﻿ if ﻿yi=max⁡kyky_i=\max_ky_kyi​=maxk​yk​﻿


Multi-layer

As seen in the OR example, this works fine for that gate. But if you would use this for an XOR gate, this technique would fail. That's why we use multiple layers: a multi-layer perceptron.


We use two consecutive steps:

  1. Calculate the position of the data point to each of the decision boundaries
  2. Combine the results of 1. to determine position of the data point to both decision boundaries and determine class


So we have at least three layers of nodes:

  • An input layer, which does no computations and has a bias
  • One or more hidden layers, which do computations and have a bias. They take as input the output of the previous nodes
  • An output layer, which only does computations. This takes as input the output of the last hidden layer

Training

Given ﻿y=wTx+by=w^Tx+by=wTx+b﻿, we try to learn the weights so that the value of ﻿yyy﻿ given ﻿xxx﻿ is correct, which is supervised learning with the true labels.


This can be done using stochastic gradient descent.


Feed-forward

The feed-forward pass goes as follows:

  • Initialize weights with a random value
  • Push input through the MLP row by row
  • Calculate and push forward the activations
  • Produce output value


Backpropagation

The backpropagation pass goes as follows:

  • Output value is compared to the label/target (this is output is first calculated using feed-forward)
  • Calculate error using a loss function
  • Error is propagated back through the network, layer by layer:
  • Update weights depending on how much they contributed to the error, thus trying to find the weights that minimize the loss function:
  • Calculate the gradients of the error function with respect to each weight
  • The gradient vector indicates the direction of the highest increase in a function, while we want the highest decrease


This iterative updating of the weights can be done as follows:

  • Recalculate the gradients at the beginning of each training iteration step
  • Stop when error ﻿&lt;θ&lt;\theta<θ﻿ or if we are at the max number of iterations
  • The general to update weights is as follows: ﻿wt←wt−1−ηδEδww_t\gets w_{t-1}-\eta\frac{\delta E}{\delta w}wt​←wt−1​−ηδwδE​﻿, where
  • ﻿η\etaη﻿ is the learning rate of the network
  • ﻿δEδw\frac{\delta E}{\delta w}δwδE​﻿ are partial derivatives of the error function ﻿EEE﻿ with respect to each weight of the array ﻿www﻿


Terminology

  • Epoch: all data used once for updating weights
  • Batch training: update weights after all training data are processed
  • Online training: update weights after each training sample
  • Learning rate: the amount that weights are updated


Combining classifiers

There are many different supervised models. If we combine different classifiers and exploit their own individual strengths we get an overall better performance than for individual classifiers.


Even if two classifiers have the same accuracy, they make errors on different patterns, so we can use complementary information that seemingly resides in the different classifiers.


The general approach for using multiple classifiers is as follows:

  • Take your input pattern
  • Run it through all your classifiers (which output class posterior probability)
  • Put all these outputs in a combiner, which gives the improved class posterior probability


This last step can be done in a bunch of different ways:


Kullback-Leibler (soft-type)

The Kullback–Leibler divergence (also called relative entropy) is a measure of how one probability distribution is different from a second, reference probability distribution. Choose ﻿P(Ci∣x)P\left(C_i\mid x\right)P(Ci​∣x)﻿, the minimum average KL distance between the probabilities.


Then assign a sample to the class that maximizes ﻿max⁡ωi∏j=1LPj(ci∣x)\max_{\omega_i}\prod_{j=1}^LP_j\left(c_i\mid x\right)maxωi​​∏j=1L​Pj​(ci​∣x)﻿ for ﻿LLL﻿ classifiers


Majority voting (hard-type)

Assign unknown pattern to class for which their is consensus: e.g. in a 2 class problem, assign a sample to the class if ﻿12⋅L+1\frac{1}{2}\cdot L+121​⋅L+1﻿ classifiers agree on that class.


This has some assumptions:

  • Number of classifiers is odd
  • Each classifier has the same probability of making a correct classification
  • Decision by each classifier is taken independently of other classifiers


The downside of this is when we are dealing with class imbalance in the training data, then there might not be a majority. Some possible rebalancing solutions are:

  • Oversample the smaller class
  • Undersample the larger class
  • Randomize
  • Focussed points near/far from the boundary


Or we can use different parameters for the two classes in the cost function.
