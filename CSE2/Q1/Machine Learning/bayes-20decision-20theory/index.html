Prior probability ﻿p(ωi)p\left(\omega_i\right)p(ωi​)﻿

The prior probability is the probability that ﻿ωi\omega_iωi​﻿ occurs, regardless of the feature vector. We can either use external knowledge to obtain this, or we can simply estimate it by using ﻿p(ωi)=NiNp\left(\omega_i\right)=\frac{N_i}{N}p(ωi​)=NNi​​﻿. This the number of observations of ﻿ωi\omega_iωi​﻿ divided by the total number of observations ﻿NNN﻿.


Class-conditional probability density function ﻿p(x∣ωi)p\left(x\mid\omega_i\right)p(x∣ωi​)﻿

This density function is the probability that ﻿xxx﻿ occurs given that ﻿xxx﻿ is member of class ﻿ωi\omega_iωi​﻿. Estimations of this function will be discussed later.


The implicit assumption that ﻿xxx﻿ can take any value has been made, in the case that feature vectors can only take discrete values, the density function becomes a probability function.


The unconditional probability ﻿p(x)p\left(x\right)p(x)﻿

This is the probability that ﻿xxx﻿ occurs, regardless of which class it belongs to. It can be calculated by calculating the total probability of ﻿xxx﻿. If we have 2 distinct classes, this would be ﻿p(x)=p(x∣ω1)p(ω1)+p(x∣ω2)p(ω2)p\left(x\right)=p\left(x\mid\omega_1\right)p\left(\omega_1\right)+p\left(x\mid\omega_2\right)p\left(\omega_2\right)p(x)=p(x∣ω1​)p(ω1​)+p(x∣ω2​)p(ω2​)﻿.


If we are comparing probabilities, this factor can be left out as we divide by this factor on both sides.

Types of classifications

We can classify a data set in two ways:

  • "Hard" classification - assign a sample to a class and output the class label
  • "Probabilistic" classification
  • Training - estimate the probability distribution over a set of classes
  • Testing - estimate the probability that a sample belongs to a class


We decide which label to assign to a class based on one or multiple decision boundaries. But it is also possible that there is a missing decision boundary, as the class could be too small that no objects are assigned to the class.


Bayes error

It could be that the probability distribution is perfectly known (we have the true distribution), then we can still have errors in our prediction. This is because you often can't predict with 100% or 0% certainty that a sample belongs to a class given a certain feature.

We want to get the probability of our to as low as possible, the Bayes error ﻿ϵ\epsilonϵ﻿:

  • the minimal attainable error (which is typically ﻿&gt;0&gt;0>0﻿).
  • in reality, we won't know the actual distribution, so this is unobtainable
  • this error depends on the distribution of the data
  • in general you can't compute the error


Minimizing risk

Sometimes misclassification of class ﻿AAA﻿ to class ﻿BBB﻿ is much more dangerous than misclassification of class ﻿BBB﻿ to class ﻿AAA﻿.


To handle this, we define a misclassification cost of misclassifying an object, thus reducing the misclassifications on one side (but this probability will increase misclassifications the other way around).


We create a loss function to balance our wishes (e.g. less misclassifications of ﻿AAA﻿ to ﻿BBB﻿ than ﻿BBB﻿ to ﻿AAA﻿) which return the price paid for inaccuracy of the classification. We try to minimise this loss function.
