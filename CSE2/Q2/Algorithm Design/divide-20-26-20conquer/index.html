Divide & Conquer

The general idea of divide & conquer goes as follows:

  1. Break a problem into smaller pieces
  2. Solve them
  3. Often recursively
  4. When small enough, solve as base case
  5. Combine the results


Remember that quicksort and mergesort are such algorithms. We can prove correctness by using induction or by expanding the base case and repeatedly unfolding.


Closest pair of points

Given a set of points ﻿SSS﻿ where every point ﻿pip_ipi​﻿ has an ﻿xxx﻿ coordinate ﻿xix_ixi​﻿ and ﻿yyy﻿ coordinate ﻿yiy_iyi​﻿. We use the Euclidean distance, what is the pair of points such that ﻿d(i,j)d\left(i,j\right)d(i,j)﻿ is minimal?


We could just brute-force this, which would give a time complexity of ﻿O(n2)O\left(n^2\right)O(n2)﻿, as we check every pair of points.


---

So we need a better way, for example:

  • Divide: draw a vertical line ﻿LLL﻿ that roughly evenly divides both left and right
  • Conquer: find the closest pair on each side
  • Combine: find the closest pair with one point on each side
  • Return the minimum of these 3


But this could be a bad solution if many points are close to the line, because if the smallest distance between the two sides is ﻿δ\deltaδ﻿, we also need to consider points in the distance of ﻿δ\deltaδ﻿ from the seperation. That is because the smallest distance may be between different points on both sides of the line!

Image taken from slides by Stefan Hugtenburg


So if many points are close to this line, we might still need to compare all, which would again yield ﻿O(n2)O\left(n^2\right)O(n2)﻿.


So we need another better solution: only comparing ﻿111111﻿ positions in the sorted list (by ﻿yyy﻿-coordinate), which would be ﻿O(nlog⁡n)O\left(n\log n\right)O(nlogn)﻿ (because of sorting).


Why ﻿111111﻿? Let ﻿sis_isi​﻿ be the point in the ﻿2δ2\delta2δ﻿-strip with the ﻿iii﻿th smallest ﻿yyy﻿-coordinate. We want to prove that if ﻿∣i−j∣&gt;11\left|i-j\right|&gt;11∣i−j∣>11﻿, then the distance between ﻿sis_isi​﻿ and ﻿sjs_jsj​﻿ is at least ﻿δ\deltaδ﻿.


---

Proof

Note that:

  • Draw boxes of ﻿0.5δ×0.5δ0.5\delta\times0.5\delta0.5δ×0.5δ﻿ around the points, no two points are in the same ﻿0.5δ×0.5δ0.5\delta\times0.5\delta0.5δ×0.5δ﻿-box. If that were the cases, our recursive call for the left/right side would have yielded a different closest pair!
  • Two points that have two rows between them have a distance ﻿≥2⋅0.5δ=δ\ge2\cdot0.5\delta=\delta≥2⋅0.5δ=δ﻿. So we only consider points at most ﻿222﻿ rows away, which means I would need to consider the following green block (which is ﻿111111﻿!):

Note that we only check the next points, not the previous ones, as we already checked those. We could decrease this ﻿111111﻿ down to ﻿555﻿, but that makes the proof much harder.

The recurrence equation for this is ﻿T(n)=2T(n2)+O(nlog⁡n)T\left(n\right)=2T\left(\frac{n}{2}\right)+O\left(n\log n\right)T(n)=2T(2n​)+O(nlogn)﻿, we could use repeated unfolding, but that would be a mess. We can also use the master method.

This will make ﻿aaa﻿ calls, in the ﻿kkk﻿th layer you have ﻿T(nbk)T\left(\frac{n}{b^k}\right)T(bkn​)﻿ calls and in the end you will have a layer with ﻿T(1)T\left(1\right)T(1)﻿:

Note that there are ﻿alog⁡bn a^{\log_bn\ }alogb​n ﻿ leaves, which means that we get ﻿Θ(alog⁡bn)=Θ(nlog⁡ba)\Theta\left(a^{\log_bn}\right)=\Theta\left(n^{\log_ba}\right)Θ(alogb​n)=Θ(nlogb​a)﻿ time complexity.


The master method states that for the previously given ﻿T(n)T\left(n\right)T(n)﻿, we get ﻿T(n)=Θ(nlog⁡ba)+∑i=0log⁡bnai⋅f(nbi)T\left(n\right)=\Theta\left(n^{\log_ba}\right)+\sum_{i=0}^{\log_bn}a^i\cdot f\left(\frac{n}{b^i}\right)T(n)=Θ(nlogb​a)+∑i=0logb​n​ai⋅f(bin​)﻿ where:

  • The first term represents the work in the leaves
  • The second term represents the work in combining the results


Note that the running time can be either:

  1. Dominated by the leaves, if ﻿f(n)f\left(n\right)f(n)﻿ is ﻿O(nlog⁡ba−ϵ)O\left(n^{\log_ba-\epsilon}\right)O(nlogb​a−ϵ)﻿, then ﻿T(n)T\left(n\right)T(n)﻿ is ﻿Θ(nlog⁡ba)\Theta\left(n^{\log_ba}\right)Θ(nlogb​a)﻿ for some ﻿ϵ&gt;0\epsilon&gt;0ϵ>0﻿
  2. Evenly distributed throughout the tree, if ﻿f(n)f\left(n\right)f(n)﻿ is ﻿Θ(nlog⁡ba)\Theta\left(n^{\log_ba}\right)Θ(nlogb​a)﻿, then ﻿T(n)T\left(n\right)T(n)﻿ is ﻿Θ(nlog⁡balog⁡n)\Theta\left(n^{\log_ba}\log n\right)Θ(nlogb​alogn)﻿
  3. Dominated by the root, if ﻿f(n)f\left(n\right)f(n)﻿ is ﻿Ω(nlog⁡ba+ϵ)\Omega\left(n^{\log_ba+\epsilon}\right)Ω(nlogb​a+ϵ)﻿, then ﻿T(n)T\left(n\right)T(n)﻿ is ﻿Θ(f(n))\Theta\left(f\left(n\right)\right)Θ(f(n))﻿ for some ﻿ϵ&gt;0\epsilon&gt;0ϵ>0﻿

  5. This will be always be satisfied by polynomials


So, the mater method will be:

  • Identify ﻿a,b,f(n)a,b,f\left(n\right)a,b,f(n)﻿ from a given recurrence equation
  • Determine ﻿nlog⁡ban^{\log_ba}nlogb​a﻿
  • Compare ﻿nlog⁡ban^{\log_ba}nlogb​a﻿ and ﻿f(n)f\left(n\right)f(n)﻿
  • Determine the right case from the master method and apply

Back to closest pair of points

We were talking about the closest pair of points problem, which had the recurrence equation (the unoptimized one for sorting every time) ﻿T(n)=2T(n2)+O(nlog⁡n)T\left(n\right)=2T\left(\frac{n}{2}\right)+O\left(n\log n\right)T(n)=2T(2n​)+O(nlogn)﻿, which means that:

  • ﻿a=2a=2a=2﻿
  • ﻿b=2b=2b=2﻿
  • ﻿f(n)=O(n)f\left(n\right)=O\left(n\right)f(n)=O(n)﻿


Thus ﻿nlog⁡ba=nlog⁡22=n1=nn^{\log_ba}=n^{\log_22}=n^1=nnlogb​a=nlog2​2=n1=n﻿, so:

  • ﻿f(n)f\left(n\right)f(n)﻿ is ﻿Θ(nlog⁡n)\Theta\left(n\log n\right)Θ(nlogn)﻿
  • ﻿f(n)f\left(n\right)f(n)﻿ is not ﻿O(n1−ϵ)O\left(n^{1-\epsilon}\right)O(n1−ϵ)﻿ , so not case 1
  • ﻿f(n)f\left(n\right)f(n)﻿ is not ﻿Θ(n)\Theta\left(n\right)Θ(n)﻿, so not case 2
  • ﻿f(n)f\left(n\right)f(n)﻿ is not ﻿Ω(n1+ϵ)\Omega\left(n^{1+\epsilon}\right)Ω(n1+ϵ)﻿, so not case 3 (as we will not find an ﻿ϵ\epsilonϵ﻿ for which ﻿nlog⁡nn\log nnlogn﻿ is a lower bound, as ﻿n1+cn^{1+c}n1+c﻿ is ﻿Ω(nlog⁡n)\Omega\left(n\log n\right)Ω(nlogn)﻿)


So, we can't use the master method here.


Instead, we can try to prove by induction. We first guess the answer and then prove it.


Matrix multiplication

We can divide a matrix into 7 multiplications of half the size and then merge it in ﻿O(n2)O\left(n^2\right)O(n2)﻿, giving ﻿T(n)=7T(n2)+O(n2)T\left(n\right)=7T\left(\frac{n}{2}\right)+O\left(n^2\right)T(n)=7T(2n​)+O(n2)﻿, which gives us ﻿Θ(nlog⁡27)\Theta\left(n^{\log_27}\right)Θ(nlog2​7)﻿﻿≈Θ(n2.81)\approx\Theta\left(n^{2.81}\right)≈Θ(n2.81)﻿ instead of ﻿Θ(n3)\Theta\left(n^3\right)Θ(n3)﻿ (which helps a lot!)
