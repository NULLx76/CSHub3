Row 000

If we take row 000, we see that the values of the hash functions are both 111. As S1S_1S1​ and S4S_4S4​ are both 111 in that row, our new signature function becomes (as 1<∞1\lt \infty1<∞)

S1S_1S1​ S2S_2S2​ S3S_3S3​ S4S_4S4​
h1h_1h1​ 111 ∞\infty∞ ∞\infty∞ 111

Row 111

Note that h1(1)=2h_1(1)=2h1​(1)=2 and h2(1)=4h_2(1)=4h2​(1)=4 and S3=1S_3=1S3​=1. So we will set the values for S3S_3S3​ in our signature matrix:

S1S_1S1​ S2S_2S2​ S3S_3S3​ S4S_4S4​
h1h_1h1​ 111 ∞\infty∞ 222 111
h2h_2h2​ 111 ∞\infty∞ 222 111


Row 222

Note that h1(1)=3h_1(1)=3h1​(1)=3 and h2(1)=2h_2(1)=2h2​(1)=2 and S2S_2S2​ and S4S_4S4​ are 111. We will only fill in these values for S2S_2S2​ as 1<3∧1<21\lt 3 \land 1 \lt 21<3∧1<2, resulting in:

S1S_1S1​ S2S_2S2​ S3S_3S3​ S4S_4S4​
h1h_1h1​ 111 333 222 111
h2h_2h2​ 111 222 222 111


Row 333

Repeat the previous steps, you'll get:

S1S_1S1​ S2S_2S2​ S3S_3S3​ S4S_4S4​
h1h_1h1​ 111 333 222 111
h2h_2h2​ 000 222 000 000


Row 444

Repeat the previous steps, you'll get:

S1S_1S1​ S2S_2S2​ S3S_3S3​ S4S_4S4​
h1h_1h1​ 111 333 000 111
h2h_2h2​ 000 222 000 000


From this we can estimate the Jaccard similarity. Note that this example is much too small for the law of large numbers to assure that estimates are close. But using this, we'd get the following estimates:
