Question 5

Why is Kernel preferred over k=1 (or vice versa)?


﻿k=1k=1k=1﻿ only takes one neighbor into account. Kernel takes all points into account in a weighted way. The closer a point is, the higher the weight. Kernel takes more data into account, and is thus better at ignoring outliers.


Question 7

Minhashing - Why do we need to create random permutations of rows?


They are used to create the signature matrix. Because of the way it is constructed, it can be used to approximate Jaccard similarity.


Question 7b

And how do we choose the number of row permutations?


In practice it is application dependent. It's a trade-off between accuracy and efficiency. (If you need to calculate it on the exam, it will be given.)

Why do we need LSH (Locality-sensitive hashing)?


To calculate similarities without having to brute force it.


Question 10

Building the signature matrix

  1. First signature matrix row: for each column: write down the first row where a 1 appears in the input matrix.
  2. All other signature matrix rows are based on the different permutations: do the same as in step 1, but go through the input rows in the order specified by the permutation.

You want to find documents similar to document 1. Check the two red boxes. In the top box, column 3 is equal to column 1, and therefore is a candidate. In the bottom box, no columns are equal to column 1. We consider document 3 similar to document 1.
