Code quality

A module can be a class, a function or higher-level software component


  • Cohesion is the degree to which methods inside a module belong together
  • High cohesion: classes and modules are independent as far as possible, which simplifies modification


  • Coupling is the degree to which one class (or module) relies on (calls or uses methods of) another class
  • Low coupling: a module should implement one single, clear, well-defined task


An average person can hold 5-9 concepts in short-term memory. Because of this, the higher the complexity, the higher the amount of bugs: we want to decrease complexity


Design patterns

Design patterns are general, reusable solutions to problems that commonly occur in software design. 


Image taken from slides by Annibale Panichella


We can classify them under a few types of patterns:

  • Behavioral patterns: Capturing the behavior among a collection of objects
  • Creational patterns: Creating objects and collections of objects in a flexible way
  • Structural patterns: Representing a collection of related objects
  • Concurrency patterns: Dealing with multi-threaded programs
  • Architectural patterns: Useful for recurring problems in software architecture within given context
