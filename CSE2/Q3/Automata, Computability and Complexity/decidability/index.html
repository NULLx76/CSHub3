Questions & decidability

  • Every question about regular languages is decidable
  • Some questions about CFLs are decidable, but some are not
  • The "Halting problem" (will a program terminate or not?) is not decidable
  • Many questions about TMs are not decidable and some are not even Turing recognizable


Halting problem

Some example questions:

  • Given a program, will it halt?
  • Given a TM, will it halt when run on some particular string?
  • Given some program written in a programming language, will it ever get into an infinite loop?


In general, we can't always know. The best we can do is running the program and see whether it halts but for many programs we can manually prove it will always halt. In general, it is undecidable.


Regular languages and decidability

DFA

Given a DFA and a string, will the DFA accept?


We know that languages are decidable, so we must express the problem in terms of a language. We can encode our problem as such: is the language ADFA={<B,w>∣B is a DFA that accepts string w}A_{DFA}=\{\text{\<}B,w\text{\>} | B \text{ is a DFA that accepts string \(w\)}\}ADFA​={<B,w>∣B is a DFA that accepts string w} decidable? Here we encode the DFA into a string, which we denote with the <> notation.


To prove this is decidable, we create a TM that decides it:

  • The TM is given as input ﻿&lt;B,w&gt;&lt;B,w&gt;<B,w>﻿ as described previously
  • The TM checks to make sure ﻿BBB﻿ is a valid representation of a DFA
  • The TM then simulates ﻿BBB﻿ on ﻿www﻿
  • If ﻿BBB﻿ reaches a final state at the end of ﻿www﻿, then the TM will accept. Otherwise it will reject
  • This TM will always halt.


NFA

The same could be proven for a NFA, either by simulating the NFA directly or converting it to a DFA (could be done by a TM) and then simulating the DFA as above.


RegEx

Deciding whether a regular expression ﻿RRR﻿ generates ﻿www﻿ is also decidable, as we can build a TM that given ﻿RRR﻿ and ﻿www﻿, will determine whether ﻿RRR﻿ generates ﻿www﻿ and this will halt.


This is because we can convert ﻿RRR﻿ into a NFA (a TM can do this), construct the string ﻿&lt;B,w&gt;&lt;B,w&gt;<B,w>﻿ and use the previous section.


Halting

Often, it is self-evident that an algorithm halts. Sometimes we should prove it carefully.


Note that proving a language is decidable involves 2 steps: prove that is recognizes the strings properly and prove that it always halts.


Questions about languages

You can ask a few questions about a languages (all of them are decidable for DFAs):

  • Acceptance testing (﻿ADFAA_{DFA}ADFA​﻿): is a string in the language? (previous section)


  • Emptiness testing (﻿EDFAE_{DFA}EDFA​﻿): is the language empty?
  • Given a DFA ﻿AAA﻿, can we go from the initial state to a final state? If so, then the DFA could generate some string
  • This is a graph problem: mark the initial state, mark any state where there is a transition to it from any marked state, until no new states get marked. Check if any final state is marked to determine whether the language is empty.


  • EQuality testing (﻿EQDFAEQ_{DFA}EQDFA​﻿): are do two DFAs accept the same language?
  • Let ﻿CCC﻿ be the symmetric difference between ﻿AAA﻿ and ﻿BBB﻿ (﻿(A∩B‾)∪(A‾∩B)\left(A\cap\overline{B}\right)\cup\left(\overline{A}\cap B\right)(A∩B)∪(A∩B)﻿ ). If ﻿A=BA=BA=B﻿, this symmetric difference will be ﻿∅\varnothing∅﻿.
  • We build a DFA ﻿CCC﻿ to accept the symmetric difference and use the TM from "emptiness" to test whether this difference is empty.


Context -Free Languages and decidability

The 3 questions about languages mentioned previously are:

  • Acceptance testing (﻿ACFGA_{CFG}ACFG​﻿): decidable
  • Emptiness testing (﻿ECFGE_{CFG}ECFG​﻿): decidable
  • EQuality testing (﻿EQCFGEQ_{CFG}EQCFG​﻿): not decidable!


Some other questions which are all not decidable:

  • Is a CFG ambiguous?
  • Do two CFGs have any strings they generate in common?
  • Is the complement of a CFG also a CFG?


Proof of decidability of acceptance testing

All grammars are parsable, we can create a parser for every language that we can use to parse a string. Some kinds of grammars can be parsed very efficiently (﻿O(N)O\left(N\right)O(N)﻿), but in general it may take ﻿O(N3)O\left(N^3\right)O(N3)﻿, where ﻿NNN﻿ is the length of the string


Idea 1: enumerate all leftmost derivations. For each, test to see if it generates ﻿www﻿. But, if ﻿www﻿ is not in the language it may not halt!

Idea 2:

  • Given inputs ﻿GGG﻿ (a CFG) and ﻿www﻿ (a string), we convert ﻿GGG﻿ into a Chomsky Normal Form.
  • Now, at each step the length grows by exactly ﻿111﻿. We also need some steps to convert variables into terminals
  • Note that every derivation has exactly ﻿2N−12N-12N−1﻿ steps. So we list all derivations of length ﻿2N−12N-12N−1﻿ and check for each derivation if it generates ﻿www﻿.
  • If any derivation generates ﻿www﻿, accept. Else reject.


Proof of decidability of emptiness testing

Idea: mark all symbols that can generate a terminal and if all rules from the starting state has a marked symbol, the language is not empty.


Formally:

  • Mark all terminal symbols
  • Repeat until nothing more can be marked:
  • Look for a rule ﻿A→XXXXA\to XXXXA→XXXX﻿ where all symbols on the right have been marked
  • Mark the nonterminal on the left side in all rules
  • If the start symbol is marked, then reject, else accept.


Halting problem

A variant of the Halting problem is the acceptance problem, which tries to accept the universal TM. We can prove that it is not decidable by contradiction:

  • Assume ﻿ATMA_{TM}ATM​﻿ is decidable and let ﻿H(⟨M,w⟩)H\left(\left\langle M,w\right\rangle\right)H(⟨M,w⟩)﻿﻿HHH﻿ be the TM that decides ﻿ATMA_{TM}ATM​﻿
  • Using ﻿HHH﻿, construct a new machine ﻿DDD﻿ that uses ﻿HHH﻿:
  • Run ﻿HHH﻿ on input ﻿⟨M,⟨M⟩⟩\left\langle M,\left\langle M\right\rangle\right\rangle⟨M,⟨M⟩⟩﻿, so we run ﻿MMM﻿ with itself as input, so we test if ﻿MMM﻿ accepts itself.
  • Output the opposite of what ﻿HHH﻿ outputs, if ﻿HHH﻿ accepts, reject. If ﻿HHH﻿ rejects, accept


So, ﻿D(⟨M⟩)D\left(\left\langle M\right\rangle\right)D(⟨M⟩)﻿ would accept if ﻿MMM﻿ does not accept ﻿⟨M⟩\left\langle M\right\rangle⟨M⟩﻿, and reject if ﻿MMM﻿ accepts ﻿⟨M⟩\left\langle M\right\rangle⟨M⟩﻿.


If we run ﻿DDD﻿ on itself, ﻿D(⟨D⟩)D\left(\left\langle D\right\rangle\right)D(⟨D⟩)﻿, we'd get the following:

  • Accept if ﻿DDD﻿ does not accept ﻿⟨D⟩\left\langle D\right\rangle⟨D⟩﻿
  • Reject if ﻿DDD﻿ accepts ﻿⟨D⟩\left\langle D\right\rangle⟨D⟩﻿


This is a paradox: ﻿DDD﻿ accepts ﻿⟨D⟩\left\langle D\right\rangle⟨D⟩﻿ if ﻿DDD﻿ does not accept ﻿⟨D⟩\left\langle D\right\rangle⟨D⟩﻿. This is a contradiction, so the acceptance problem is not decidable.


A non-Turing recognizable language

  • A language is co-Turing recognizable if its complement ﻿L‾\overline{L}L﻿ is Turing recognizable
  • If a language ﻿LLL﻿ is decidable, then ﻿LLL﻿ is Turing recognizable and co-Turing recognizable.
  • If a language ﻿LLL﻿ is Turing recognizable an co-Turing recognizable, then ﻿LLL﻿ is decidable. We can prove this as such:
  • Let ﻿M1M_1M1​﻿ and ﻿M2M_2M2​﻿ be recognizers for ﻿LLL﻿ and ﻿L‾\overline{L}L﻿ respectivel
  • Run both in parallel, ﻿xxx﻿ is in either ﻿LLL﻿ or ﻿L‾\overline{L}L﻿, so either will eventually halt and one will accept
  • If ﻿M1M_1M1​﻿ accepts, then accept. If ﻿M2M_2M2​﻿ accept, then reject. You'll always halt.


We have shown that ﻿ATMA_{TM}ATM​﻿ is Turing recognizable, but not decidable. So ﻿ATM‾\overline{A_{TM}}ATM​​﻿ is not Turing recognizable.


Reducibility

Using reducibility we can prove that some problems are undecidable. We do that by "reducing" a hard problem into another, easier, problem.


If we can reduce ﻿AAA﻿ to ﻿BBB﻿, then:

  • A way to solve ﻿BBB﻿ yields a way to solve ﻿AAA﻿
  • If ﻿AAA﻿ is unsolvable, then ﻿BBB﻿ is also unsolvable and vice versa.


If we can reduce ﻿AAA﻿ to ﻿BBB﻿, we denote it as ﻿A≥BA\ge BA≥B﻿, meaning ﻿AAA﻿ is more or as difficult as ﻿BBB﻿.


There are three forms of reduction:

  1. Direct reduction
  2. Reduction via computation histories
  3. Reduction via mappings


Logic

Known: ﻿ATMA_{TM}ATM​﻿ is undecidable


Now we want to prove that ﻿PPP﻿ is undecidable. We can prove that as such:

  • Assume ﻿PPP﻿ is decidable
  • We reduce ﻿ATMA_{TM}ATM​﻿ (hard problem) into ﻿PPP﻿ (easier problem)
  • Use the solution of ﻿PPP﻿ to solve ﻿ATMA_{TM}ATM​﻿:
  • Use the decidability of ﻿PPP﻿ to find an algorithm to decide ﻿ATMA_{TM}ATM​﻿
  • Build a ﻿TMTMTM﻿ to decide ﻿ATMA_{TM}ATM​﻿ using the TM to decide ﻿PPP﻿ as a subroutine
  • But we know that a decider for ﻿ATMA_{TM}ATM​﻿ does not exist, so ﻿PPP﻿ is not decidable.


Reducing the Halting problem

﻿HALTTM={⟨M,w∣M is a TM and M halts on w}HALT_{TM}=\left\{\left\langle M,w\right|M\ is\ a\ TM\ and\ M\ halts\ on\ w\right\}HALTTM​={⟨M,w∣M is a TM and M halts on w}﻿ is undecidable.


Proof:

  • Assume ﻿HALTTMHALT_{TM}HALTTM​﻿ is decidable, so there is a TM ﻿RRR﻿ that decides ﻿HALTTMHALT_{TM}HALTTM​﻿
  • Use ﻿RRR﻿ to build another TM ﻿SSS﻿ that decides ﻿ATMA_{TM}ATM​﻿
  • But ﻿ATMA_{TM}ATM​﻿ is undecidable: contradiction


So we have two TM's:

  • ﻿RRR﻿, an algorithm to decide ﻿HALTTMHALT_{TM}HALTTM​﻿
  • ﻿SSS﻿, an algorithm to decide ﻿ATMA_{TM}ATM​﻿


We construct this ﻿SSS﻿ using ﻿RRR﻿ as such:

  • Run ﻿RRR﻿ on ﻿⟨M,w⟩\left\langle M,w\right\rangle⟨M,w⟩﻿ to see if ﻿MMM﻿ halts or loops
  • If ﻿RRR﻿ rejects, it means ﻿MMM﻿ loops. Then reject
  • If ﻿RRR﻿ accepts, it means ﻿MMM﻿ halts, so simulate ﻿MMM﻿ on input ﻿www﻿.


Reducing "does a TM accept any string?"

﻿ETM={⟨M∣M is a TM and L(M)=∅}E_{TM}=\left\{\left\langle M\right|M\ is\ a\ TM\ and\ L\left(M\right)=\varnothing\right\}ETM​={⟨M∣M is a TM and L(M)=∅}﻿ is undecidable.


We prove this in the same way as last time.


So we again have two TM's:

  • ﻿RRR﻿, an algorithm to decide ﻿ETME_{TM}ETM​﻿
  • ﻿SSS﻿, an algorithm to decide ﻿ATMA_{TM}ATM​﻿ (which gets ﻿⟨M,w⟩\left\langle M,w\right\rangle⟨M,w⟩﻿ as input)

  • Assume that ﻿www﻿ is a constant and our input is ﻿xxx﻿. We will reject all strings that do not match ﻿www﻿.
  • It may or may not accept ﻿www﻿, so ﻿L(M′)={w}L\left(M&#x27;\right)=\left\{w\right\}L(M′)={w}﻿ (if ﻿MMM﻿ accepts ﻿www﻿) or ﻿L(M′)=∅L\left(M&#x27;\right)=\varnothingL(M′)=∅﻿ (otherwise)

  • Use ﻿RRR﻿ to decide whether ﻿L(M′)L\left(M&#x27;\right)L(M′)﻿ is empty or not:

Rice's theorem

If ﻿PPP﻿ is a set of Turing Machines satifsying (or ﻿PPP﻿ is a property of languages):

  1. ﻿PPP﻿ is exclusively defined in terms of input/output behavior of TMs. If ﻿L(M1)=L(M2)L\left(M_1\right)=L\left(M_2\right)L(M1​)=L(M2​)﻿, then ﻿⟨M1⟩∈P⇔⟨M2⟩∈P\left\langle M_1\right\rangle\in P\Leftrightarrow\left\langle M_2\right\rangle\in P⟨M1​⟩∈P⇔⟨M2​⟩∈P﻿
  2. ﻿PPP﻿ is not trivial, ﻿P≠∅P\ne\varnothingP​=∅﻿ and ﻿P‾≠∅\overline{P}\ne\varnothingP​=∅﻿

Then ﻿PPP﻿ is not decidable.
