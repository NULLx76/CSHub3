Complexity of problems

The complexity of a problem is the worst-case complexity of a best algorithm for the problem, when executed on a pre-determined machine model (i.e. multitape TM, non deterministic TM etc):

  • We define "best algorithm" as follows: ﻿AAA﻿ is the best algorithm for ﻿XXX﻿ if for every algorithm ﻿BBB﻿ for ﻿XXX﻿ and for all ﻿nnn﻿ it holds that ﻿TA(n)≤TB(n)T_A\left(n\right)\le T_B\left(n\right)TA​(n)≤TB​(n)﻿.
  • This pre-determined machine model will be a 1-tape DTM. Note that for every ﻿kkk﻿-tape (﻿k≥2k\ge2k≥2﻿) DTM solving a problem ﻿XXX﻿ in ﻿t(n)t\left(n\right)t(n)﻿ time, there is a single-tape DTM that solves ﻿XXX﻿ in ﻿O(t2(n))O\left(t^2\left(n\right)\right)O(t2(n))﻿ time. So for a multitape TM we only have quadratic overhead.


Strong Church-Turing Thesis (SCTT)

Every realisable algorithmic process can be simulated on a 1-tape DTM with polynomial overhead. So this implies that ALL machine models are polynomial equivalent (tablet, phone, pc, etc.)


This is accepted, but not proven. Quantum computing might be an exception to this rule.

Grouping classes

Polynomial

An algorithm ﻿AAA﻿ with time complexity ﻿f(n)f\left(n\right)f(n)﻿ is called polynomial-time if ﻿f(n)f\left(n\right)f(n)﻿ is ﻿O(nc)O\left(n^c\right)O(nc)﻿ for some constant ﻿c&gt;0c&gt;0c>0﻿. This is sometimes written as ﻿f(n)=nO(1)f\left(n\right)=n^{O\left(1\right)}f(n)=nO(1)﻿.


P

Now we can define our first class, ﻿PPP﻿ (polynomial):

P=∪k≥0TIME(nk)={L∣L is decided by a DTM in polynomial time}=set of all languages decidable in polynomial time\begin{aligned} P &= \cup_{k\geq 0}\text{TIME}(n^k) \\ &= \{L | L \text{ is decided by a DTM in polynomial time}\} \\ &= \text{set of all languages decidable in polynomial time} \end{aligned}P​=∪k≥0​TIME(nk)={L∣L is decided by a DTM in polynomial time}=set of all languages decidable in polynomial time​


As the SCTT only adds polynomial overhead, a problem in ﻿PPP﻿ will remain in ﻿PPP﻿, even if you were to change the machine.


Tractable

  • A problem ﻿XXX﻿ is tractable (easy) if ﻿X∈PX\in PX∈P﻿
  • A problem ﻿XXX﻿ is intractable (hard) of ﻿X∉PX\notin PX∈/​P﻿


We call tractable problems easy because a faster machine makes a lot of difference for problems in ﻿PPP﻿, but for problems not in ﻿PPP﻿ it barely makes a difference.


NP

We can also define our second class, ﻿NPNPNP﻿ (non-deterministic polynomial):

NP=∪k≥0NTIME(nk)={L∣L is decided by a NTM in polynomial time}\begin{aligned} NP &= \cup_{k\geq 0}\text{NTIME}(n^k) \\ &= \{L | L \text{ is decided by a NTM in polynomial time}\} \end{aligned}NP​=∪k≥0​NTIME(nk)={L∣L is decided by a NTM in polynomial time}​


Note that the only difference between ﻿PPP﻿ and ﻿NPNPNP﻿ is the fact that ﻿NPNPNP﻿ can use a non-deterministic TM.


But, simulating a ﻿NPNPNP﻿ on a 1-tape TM does take exponential time, as we need to try every computation path. If we are given a solution found by a NTM, we can verify this solution in deterministic polynomial time.


Now, as ﻿PPP﻿ is a subset of ﻿NPNPNP﻿, we can compute ﻿NP−PNP-PNP−P﻿. The solution of this would contain the problems that we can't calculate in ﻿PPP﻿ but we can check their solutions in deterministic polynomial time.


P vs NP

There are many problems in ﻿NP−PNP-PNP−P﻿ in computer science.


For example: vertex cover, given a graph ﻿G=(V,E)G=\left(V,E\right)G=(V,E)﻿ and a positive integer ﻿kkk﻿. Right now, we think this is a problem in ﻿NP−PNP-PNP−P﻿, as we have not found an algorithm in polynomial time.
