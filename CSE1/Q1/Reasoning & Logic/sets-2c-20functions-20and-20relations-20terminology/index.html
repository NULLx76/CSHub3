Terms

Sets (4.1 & 4.2)

A set is a collection of elements. A set is defined entirely by the elements that it contains. An element can be anything, including another set. A set does not allow for duplicates.


Types of sets:

  • Power set: The set containing all the possible subsets of a set. The powerset of set ﻿AAA﻿ is notated as ﻿℘(A)\wp\left(A\right)℘(A)﻿. The definition is ﻿℘(A)={X ∣ X⊆A}\wp\left(A\right)=\left\{X\ \mid\ X\subseteq A\right\}℘(A)={X ∣ X⊆A}﻿. For example, if set ﻿A = {a, b}A\ =\ \left\{a,\ b\right\}A = {a, b}﻿, ﻿℘(A) = {∅, {a}, {b}, {a, b}}\wp\left(A\right)\ =\ \left\{\varnothing,\ \left\{a\right\},\ \left\{b\right\},\ \left\{a,\ b\right\}\right\}℘(A) = {∅, {a}, {b}, {a, b}}﻿. The amount of items in the powerset of set ﻿AAA﻿ can be calculated by ﻿∣℘(A)∣=2∣A∣\left|\wp\left(A\right)\right|=2^{\left|A\right|}∣℘(A)∣=2∣A∣﻿, where ﻿∣ ∣\left|\ \right|∣ ∣﻿ stands for the cardinality, size, of the set.


  • Empty set: The set containing no elements, notated as ﻿∅\varnothing∅﻿. The empty set is a subset of every set. The powerset of the empty set is not an empty set, as it contains one set: the set containing the empty set. ﻿℘(∅) = {∅} = {{}}\wp\left(\varnothing\right)\ =\ \left\{\varnothing\right\}\ =\ \left\{\left\{\right\}\right\}℘(∅) = {∅} = {{}}﻿.

The name of the variable, ﻿xxx﻿ , is arbitrary, so the same set could be denoted with many other letters. This notation can be read as "the set of ﻿xxx﻿ such that ﻿P(x)P\left(x\right)P(x)﻿"

Functions (4.4)

A function is mathematical object that expresses a functional relationship between two sets. The notation ﻿f: A→Bf:\ A\to Bf: A→B﻿ expresses the fact that f is a function from the set ﻿AAA﻿ to the set ﻿BBB﻿ . This can also be notated as ﻿BAB^ABA﻿.


  • Mapping / functional relationship: A functional relationship is a relationship between two sets, which associates exactly one element from the second set to each element of the first set.


  • Composition: If you get from set ﻿AAA﻿ to set ﻿BBB﻿ using a function (﻿fff﻿), and using another function (﻿ggg﻿) you get from set ﻿BBB﻿ to set ﻿CCC﻿, you can create a function which goes from set ﻿AAA﻿ to set ﻿CCC﻿. That function would be called a composition of two functions, in this case of ﻿ggg﻿ with ﻿fff﻿. This is notated in the following way: ﻿(g∘f)\left(g\circ f\right)(g∘f)﻿. This gives us the following: ﻿(g∘f)(a) = g(f(a))\left(g\circ f\right)\left(a\right)\ =\ g\left(f\left(a\right)\right)(g∘f)(a) = g(f(a))﻿.


  • Cross product / Cartesian product: If ﻿AAA﻿ and ﻿BBB﻿ are sets, then we can form the set ﻿A×BA\times BA×B﻿ which is defined by ﻿{A×B = {(a,b)∣a∈A and b∈B}\left\{A\times B\ =\ \left\{\left(a,b\right)\right|a\in A\ and\ b\in B\right\}{A×B = {(a,b)∣a∈A and b∈B}﻿. This set is called the cross product or Cartesian product of the sets ﻿AAA﻿ and ﻿BBB﻿.
  • A Cartesian product is created from an ordered pair, which means that in﻿(a, b)\left(a,\ b\right)(a, b)﻿, the order does actually matter. ﻿(a, b)≠(b, a)\left(a,\ b\right)\ne\left(b,\ a\right)(a, b)̸​=(b, a)﻿. The cardinality of the cartesian product ﻿A×BA\times BA×B﻿ is ﻿∣A×B∣=∣A∣⋅∣B∣\left|A\times B\right|=\left|A\right|\cdot\left|B\right|∣A×B∣=∣A∣⋅∣B∣﻿.


  • Properties of functions: Suppose that ﻿f: A→Bf:\ A\to Bf: A→B﻿ is a function from the set ﻿AAA﻿ to the set ﻿BBB﻿.
  • Domain: Set ﻿AAA﻿ of the function
  • Range: Set ﻿BBB﻿ of the function
  • Image: The set ﻿{b∈B ∣ ∃a∈A (b=f(a))}\left\{b\in B\ \mid\ \exists a\in A\ \left(b=f\left(a\right)\right)\right\}{b∈B ∣ ∃a∈A (b=f(a))}﻿ , simply said: ﻿{f(a) ∣ a∈A}\left\{f\left(a\right)\ \mid\ a\in A\right\}{f(a) ∣ a∈A}﻿. That is, the image is the set of all values, ﻿f(a)f\left(a\right)f(a)﻿, of the function, for all ﻿a∈Aa\in Aa∈A﻿ .


Function types:

  • Onto / surjective: A function ﻿f: A→Bf:\ A\to Bf: A→B﻿ is said to be onto (or surjective) if every element of ﻿BBB﻿ is equal to ﻿f(a)f\left(a\right)f(a)﻿ for some element of ﻿AAA﻿. In terms of logic, ﻿fff﻿ is onto if and only if: ﻿∀b∈B (∃a∈A(b=f(a)))\forall b\in B\ \left(\exists a\in A\left(b=f\left(a\right)\right)\right)∀b∈B (∃a∈A(b=f(a)))﻿. This means that for every element ﻿b∈Bb\in Bb∈B﻿, there is an element in set ﻿AAA﻿ for which ﻿a∈A, f(a)=ba\in A,\ f\left(a\right)=ba∈A, f(a)=b﻿. Easier said: every value in ﻿BBB﻿ is mapped by some value of ﻿AAA﻿.


  • One-to-one / injective: A function ﻿f: A→Bf:\ A\to Bf: A→B﻿ is said to be one-to-one (or injective) if and only if: ﻿∀x∈A, ∀y∈A (x≠y→f(x)≠f(y))\forall x\in A,\ \forall y\in A\ \left(x\ne y\to f\left(x\right)\ne f\left(y\right)\right)∀x∈A, ∀y∈A (x̸​=y→f(x)̸​=f(y))﻿, or ﻿∀x∈A, ∀y∈A(f(x)=f(y)→x=y)\forall x\in A,\ \forall y\in A\left(f\left(x\right)=f\left(y\right)\to x=y\right)∀x∈A, ∀y∈A(f(x)=f(y)→x=y)﻿. This means that for every value of ﻿b∈Bb\in Bb∈B﻿ , there can be only one item in the set A for which ﻿a∈A, f(a)=ba\in A,\ f\left(a\right)=ba∈A, f(a)=b﻿ (but not every element in ﻿BBB﻿ has to be mapped by ﻿AAA﻿). Easier said: there can be only one value of ﻿AAA﻿ which maps to a specific value in ﻿BBB﻿.


  • Bijective: If a function is both an onto and one-to-one, it is bijective. This means that for every value ﻿b∈Bb\in Bb∈B﻿, there will be a single value of ﻿a∈Aa\in Aa∈A﻿ for which goes: ﻿f(a)=bf\left(a\right)=bf(a)=b﻿.


Infinity (4.6)

  • Finite: A set ﻿AAA﻿ is said to be finite if there is a one-to-one correspondence (bijective) between ﻿AAA﻿ and ﻿NnN_nNn​﻿ (﻿NNN﻿ is a set of natural numbers) for some natural number ﻿nnn﻿).
  • Infinite: A set ﻿BBB﻿ is said to be infinite if there is no one-to-one correspondence (bijective) between ﻿BBB﻿ and ﻿NnN_nNn​﻿ (﻿NNN﻿ is a set of natural numbers) for some natural number ﻿nnn﻿).
  • Some statements about cardinality of finite sets:

Countable vs uncountable:

  • Countably infinite: if set ﻿AAA﻿ has a one-to-one correspondence between ﻿N \mathbb{N}\ N ﻿and ﻿AAA﻿. A set is countable when it is finite or countably infinite. The idea of ‘countable infinity’ is that even though a countably infinite set cannot be counted in a finite time, we can imagine counting all the elements of ﻿AAA﻿, one-by-one, in an infinite process. Every countably infinite set is equally large. The following sets are countably infinite:
  • ﻿N \mathbb{N}\ N ﻿
  • ﻿Z\mathbb{Z}Z﻿
  • ﻿Q \mathbb{Q}\ Q ﻿


  • Uncountably infinite: A set is uncountably infinite if it is an infinite set that contains too many elements to be countable. ﻿R \mathbb{R}\ R ﻿is uncountable as it is not possible to make an infinite list that contains every real number


Relations (4.7)

A relation on two sets, ﻿AAA﻿ and ﻿BBB﻿, is defined to be a subset of﻿A×BA\times BA×B﻿. If ﻿AAA﻿ is a set and ﻿RRR﻿ is a relation on the sets ﻿AAA﻿ and ﻿AAA﻿, then ﻿RRR﻿ is said to be a binary relation on ﻿AAA﻿. You can also have a ternary relation, with three sets. This you call the arity of a predicate. If you have a ternary relation on ﻿N \mathbb{N}\ N ﻿, the relation will be a subset of ﻿N ×N ×N = N3\mathbb{N}\ \times\mathbb{N}\ \times\mathbb{N}\ =\ \mathbb{N}^3N ×N ×N = N3﻿. An example of such a relation can be defined as follows: ﻿R = {(x, y, z)∈N 3 ∣ x − y &gt; z}R\ =\ \left\{\left(x,\ y,\ z\right)\in\mathbb{N}\ ^3\ \mid\ x\ -\ y\ &gt;\ z\right\}R = {(x, y, z)∈N 3 ∣ x − y > z}﻿. Then ﻿R(x, y, z ∈N) = {(10, 1, 1), (8, 4, 3).....}R\left(x,\ y,\ z\ \in\mathbb{N}\right)\ =\ \left\{\left(10,\ 1,\ 1\right),\ \left(8,\ 4,\ 3\right).....\right\}R(x, y, z ∈N) = {(10, 1, 1), (8, 4, 3).....}﻿, which is a subset of ﻿N3\mathbb{N}^3N3﻿.


Relations:

  • Reflexive relation: ﻿RRR﻿ is said to be reflexive if ﻿∀a ∈A (aRa)\forall a\ \in A\ \left(aRa\right)∀a ∈A (aRa)﻿ (can also be notated as ﻿∀a ∈A (R(a, a))\forall a\ \in A\ \left(R\left(a,\ a\right)\right)∀a ∈A (R(a, a))﻿). That is, a binary relation on a set is reflexive if every element of the set is related to itself. A few example of reflexive relations are ﻿===﻿ and ﻿≤\le≤﻿. ﻿&lt;&lt;<﻿ is not reflexive, as ﻿1&lt;11&lt;11<1﻿ is false.


  • Transitive relation: ﻿RRR﻿ is said to be transitive if ﻿∀a, ∀b, ∀c∈A ((aRb ∧ bRc)→(aRc))\forall a,\ \forall b,\ \forall c\in A\ \left(\left(aRb\ \wedge\ bRc\right)\to\left(aRc\right)\right)∀a, ∀b, ∀c∈A ((aRb ∧ bRc)→(aRc))﻿. This can be compared to the Law of Syllogism, as discussed in the propositional logic part of the textbook. An example of a transitive relation is ﻿&lt;&lt;<﻿, as ﻿1&lt;2&lt;31&lt;2&lt;31<2<3﻿ implies ﻿1&lt;31&lt;31<3﻿.


  • Symmetric relation: ﻿RRR﻿ is said to be symmetric if ﻿∀a∈A, ∀b∈B (aRb→bRa)\forall a\in A,\ \forall b\in B\ \left(aRb\to bRa\right)∀a∈A, ∀b∈B (aRb→bRa)﻿. That is, whenever ﻿aaa﻿ is related to ﻿bbb﻿, it follows that ﻿bbb﻿ is related to ﻿aaa﻿. An example of ﻿aaa﻿ symmetric relation is ﻿===﻿, as ﻿2=2 ∧ 2=22=2\ \wedge\ 2=22=2 ∧ 2=2﻿.


  • Antisymmetric relation: ﻿RRR﻿ is said to be antisymmetric if ﻿∀a∈A, ∀b∈B ((aRb ∧ bRa)→a=b)\forall a\in A,\ \forall b\in B\ \left(\left(aRb\ \wedge\ bRa\right)\to a=b\right)∀a∈A, ∀b∈B ((aRb ∧ bRa)→a=b)﻿. The relation ﻿RRR﻿ is antisymmetric if for any two distinct elements ﻿xxx﻿ and ﻿yyy﻿ of ﻿AAA﻿, we can’t have both ﻿xRy ∧ yRxxRy\ \wedge\ yRxxRy ∧ yRx﻿. Formulated differently: ﻿R(a, b) ∧ a ≠b→¬R(a, b)R\left(a,\ b\right)\ \wedge\ a\ \ne b\to\neg R\left(a,\ b\right)R(a, b) ∧ a ̸​=b→¬R(a, b)﻿.


Orders:

  • Partial order: A binary relation that is reflexive, antisymmetric and transitive. An example of such a relation is ﻿≤\le≤﻿. Another example of a partial order is the subset relation, ﻿⊆\subseteq⊆﻿, on the power set of any set.


  • Total order: A binary relation that is a partial order and furthermore for any two elements ﻿aaa﻿ and ﻿bbb﻿ of ﻿AAA﻿, either ﻿aRbaRbaRb﻿ or ﻿bRabRabRa﻿. An example of such a relation is ﻿≤\le≤﻿.


Partitions and equivalence relations:

  • Partition: A partition of ﻿AAA﻿ is defined to be a collection of non-empty subsets of ﻿AAA﻿ such that each pair of distinct subsets in the collection is disjoint and the union of all the subsets in the collection is ﻿AAA﻿.


  • Equivalence relation: A binary relation that is reflexive, symmetric, and transitive. Any relation defined from a partition is an equivalence relation.


  • Equivalence class: Let ﻿a∈Aa\in Aa∈A﻿. We define the equivalence class of ﻿aaa﻿ under the equivalence relation ﻿RRR﻿ to be the subset ﻿[a]R\left[a\right]_R[a]R​﻿ defined as ﻿[a]R={b∈A ∣ bRa}\left[a\right]_R=\left\{b\in A\ \mid\ bRa\right\}[a]R​={b∈A ∣ bRa}﻿. That is, the equivalence class of ﻿aaa﻿ is the set of all elements of ﻿AAA﻿ that are related to ﻿aaa﻿. See the first StackExchange link in the resources list for a great example. An equivalence class forms a partition, it is a form of a partition which has a logical structure and isn't arbitrarily chosen. For example, with the following set ﻿A = {0, 1, 2, 3}A\ =\ \left\{0,\ 1,\ 2,\ 3\right\}A = {0, 1, 2, 3}﻿ you could create your own partitions ﻿{0, 1}\left\{0,\ 1\right\}{0, 1}﻿ and ﻿{2, 3}\left\{2,\ 3\right\}{2, 3}﻿ but these combinations don't necessarily have to make sense. If the partition is an equivalence class, there is an actual reason for why the combinations are chosen, see the example down below as referenced to before.


  • Transitive closure: A relation might not be transitive, but we can use multiple steps to get to our result. This relation is called a transitive closure and is often notated as ﻿R∗R^{\ast}R∗﻿. The transitive closure is the smallest relation on a set ﻿XXX﻿ such that ﻿R ∗R^{\ \ast}R ∗﻿ contains ﻿RRR﻿ and ﻿R∗R^{\ast}R∗﻿ is minimal.
