About JavaScript

JavaScript is an implementation of EcmaScript (by far the most popular one) and the course ignores a great update of EcmaScript (ES6, released in 2015). There are many more much better languages which compile to JS, but they are only mildly spoken about in the course.

Scripting

Server-side scripting refers to scripts that run on the web server (in contrast to the client). This means that the server receives requests from client and responds to them accordingly. The server has control over the database and much more. The server is the source of truth, not the client.


Client-side scripting refers to code that is executed on the client, which mostly has to do with UI and state management (in modern applications). On the client you would place the script tag either in the HEAD (so it's loaded instantly) or down in the body (so it's loaded after the rest is loaded) depending on your script.


JavaScript

Here are a few things which are quite handy to know about JavaScript:


Builtin types/objects (the same in this case)

  1. Object (hashmap)
  2. List
  3. String
  4. Number (always double! no ints)
  5. function


Singleton objects (only declared once):

  1. false - autocast to 0
  2. true - autocast to 1
  3. undefined
  4. null
  5. NaN - subclass of number (not a number)
  6. Inf - subclass of number (Infinity)


Truthi- and Falsiness:

  1. Some objects in JavaScript can be automatically casted to other types of objects.
  2. A positive nonzero int can be used as a boolean true
  3. A number zero can be used as a boolean false
  4. An empty container (list, string, object) is falsy (so not false!)
  5. A container with elements in it is truthy (so not true!)


Casting:

  1. The + operator tries to cast either to a number.
  2. The ! operator tries to cast to a boolean


Examples:

if([1,2,3]) {console.log("reached")} // truthy: non-empty container

if([]) {console.log("never reached")} // falsy: empty container

![] == false // (! casts to boolean)

!![] == true // (! casts to boolean, then inverted with !)

[] + [] == "" // (at least one side is a string or list so its casted to a string)

"" + 1 == "1" // (at least one side is a string or list so its casted to a string)

!![] + !![] == 2 // (booleans can be casted to integers under addition)



Scoping, hoisting and this

Scoping: the context in which values and expressions are visible. JavaScript has a few scopes:

  1. local
  2. global
  3. block


A block is used to group a number of statements together with a pair of curly brackets{...}.

The scopes of values and expressions depend on where and how they are declared:

  1. var declared within a function: local scope;
  2. var declared outside of a function: global scope;
  3. novar: global scope (no matter where declared) DON'T DO THIS;
  4. let was introduced in ES6: block scope;
  5. constwas introduced in ES6: block scope, no reassignment or redeclaration (but the originally assigned element can change).


Always use let and const :)


Read more about this here


Hoisting: declarations are processed before any code is executed. If you set a variable to be a function, then you only declare it when you reach that line, so it's different than declaring a function 'normally'. So:

  1. Declarations are hoisted to the top.
  2. Expressions are not hoisted.


Read more about this here


this: in JavaScript what this refers to is dependent on how the function containing this was called. We can also set what the this scope of the function is using the bind function.

JavaScript design patterns

In JavaScript, functions are first-class citizens of the language. This means that functions can be passed as parameters, they can be returned from functions and they can be assigned to a variable. In JavaScript. functions are also object.


You can construct JavaScript object in a few different ways (This won't contain code examples, see here for code examples):

  1. Object creation with new: You can create an empty object and add fields and methods to this object, which you can later call and use
  2. Object literals: An object literal is a list of zero or more pairs of property names and associated values of an object, enclosed in curly braces


DISCLAIMER: These constructor patterns are outdated and even in ES6 you have class syntax in JavaScript. To allow for encapsulation, TypeScript is a solution which you should definitely use instead of JavaScript. So the following is unnecessary really


Basic constructor design pattern: You can create an instance of a function, which then internally has its own this scope. This scope can contain fields or methods, which can be functions as well. As you use JavaScript, you can add add properties and methods on-the-fly as it's a prototype-based language. This is horrible as it can lead to many, many mistakes. Don't do this unless you really must.


Prototype based constructor: Inheritance through prototyping. Objects have a point to another object, the object's prototype. And thus, when creating for instance an object with a basic constructor as just seen, the properties of the constructor's prototype are also accessible in the new object. If a property is not defined in the object, the prototype chain is followed. Changes to the prototype are also reflected in existing objects. See the transcript for an example.


Module pattern: Does not declare any global variables or functions unless required. It also emulates encapsulation. It uses an Immediately Invoked Function Expression. This module returns an object literal with the methods on them. You can't directly access these variables, thus it sort of emulates a private field.
