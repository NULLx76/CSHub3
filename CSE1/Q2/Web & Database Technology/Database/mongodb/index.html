MongoDB

Some information

  • An open source, document oriented database
  • BSON - Binary JSON format with dynamic schemas
  • Replication using master-slave, automatic data sharding, highly scalable
  • But no:
  • Complex queries
  • Multi-document transactions
  • ACID guarantees


JSON in MongoDB

  • Keys can include any Unicode character except for '.' and '$' (in MongoDB)
  • In MongoDB, key/value pairs are ordered because it is saved in binary format


Data model

  • Collection of documents which usually have a similar structure
  • Document is one JSON object of arbitrary complexity:
  • Belongs to one collection
  • Max 16 MB
  • Has an identifier
  • Unique within a collection
  • Immutable
  • Of any type other than Array
  • ObjectId: special 12-byte BSON
  • Small, likely unique, fast to generate


Denormalization

  • Related data in single structure with subdocuments (single operation for read/write)
  • You can choose between embedding (denormalized) and references (more flexibility, but no integrity):
  • Manual reference: references in the same collection
  • DBRef reference: references across collections and dbs
